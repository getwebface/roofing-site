## 2024-03-02 - Continuous Event Performance Optimizations
**Learning:** Attaching continuous event listeners like `pointermove` directly to sections can block the main thread and impact scrolling/rendering performance because they fire continuously and execute synchronously (calculating distance mathematically on every frame). In an environment tracking telemetry across many sections, this is a significant bottleneck. On the other hand, micro-optimizations like caching static `NodeList` results from `querySelectorAll` offer negligible speed benefits but introduce significant risks of using stale DOM nodes when the DOM mutates.
**Action:** Use `requestAnimationFrame` to throttle continuous event handlers like `pointermove` to only execute once per animation frame, drastically reducing CPU load during active movement without losing meaningful telemetry fidelity. Avoid caching static DOM query results unless they are executed repeatedly in a hot path (like an animation loop).
